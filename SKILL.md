# Skill Creptoout

适用于对 PPTX 做业务脱敏，重点处理经营结果数值、图表展示值、客户名称，同时尽量保留时间、序号、路径编号、分页和能力体系编号。

## 何时使用

- 用户要求把 PPT 数字脱敏为 `xxx`
- 用户要求保留时间、目录编号、路径编号、分页等标识
- 用户要求图表中的结果值同步脱敏
- 用户要求客户名称、客户公司名、头部客户名单脱敏
- 用户希望把当前脱敏逻辑沉淀为可复用的独立程序

## 目录结构

```text
.trae/skills/skill_creptoout/
  SKILL.md
  skill_creptoout.py
```

## 技能说明

- `skill_creptoout.py` 是独立可运行脚本，不依赖项目根目录里的 `ppt_numbers_to_xxx.py`
- 脚本直接处理 `.pptx` 包内 XML
- 文本脱敏和图表展示值脱敏同时支持
- 规则按“保留标识符，脱敏经营结果值”的口径设计

## 最新规则沉淀

- 标题保留：
  - 目录页标题、章节标题、图表标题、首页封面标题不脱敏
  - 章节编号如 `1.3`、`1.4`、`2.2` 视为标题前缀保留
  - 语义标题如 `重点指标`、`收入结构`、`竞争情况`、`工作部署`、`整体回顾` 不脱敏
- 工作举措保留：
  - 工作方法、工作要求、行动口径中的数字默认保留
  - 如 `三个50%`、`1个原则`、`2大运营`、`3大主业`、`4项保障`、`10项重点工作`
  - 如 `1个关键人+1个联系人`、`6+N`、`5G`、`2G`
- 目标值与结果值脱敏：
  - 收入、增幅、份额、占比、贡献、回款、净值、商机、规模、完成率、到达值、目标值默认脱敏
  - 文中目标值、结果值、同比值、排名值、公开市场份额、签约金额等经营结果均脱敏
  - 括号中的历史结果值和说明性结果值一并脱敏
- 表格勾稽脱敏：
  - 当正文中的目标值或结果值已经脱敏时，同页表格中可反推出该结果的配套数值同步脱敏
  - `KPI`、`各地市完成情况`、`智算项目`、`劳动竞赛`、`千万大单` 等结果表按整表或联动逻辑处理
  - 图表中的折线、柱状图、数据标签和表格中的对应结果值同步脱敏
- 客户名称脱敏：
  - 表格里一旦属于客户名称列，按整列口径统一脱敏
  - 项目清单、客户清单、友商信息、公司主体名称按客户名称规则脱敏

## 默认保留

- 时间标签：如 `2025年`、`11月`、`1-11月`
- 目录序号：如 `01`、`02`、`03`
- 路径编号：如 `路径6`
- 分页标识：如 `1/3`、`2/3`、`3/3`
- 能力体系/术语：如 `6+N`、`5G`
- 工作举措：如 `1个关键人+1个联系人`
- 标题编号：如 `1.3`、`1.4`、`2.2`
- 标题文字：如 `重点指标`、`收入结构`、`标品与ICT发展平稳`

## 默认脱敏

- 收入、增幅、占比、贡献、份额、规模、金额、回款、商机、员工人数、完成率
- 图表数据标签中的结果值
- 客户名称和客户主体名称
- 括号中的历史结果数据，如 `25年11.4%`
- 同页表格中能反推出正文结果值的配套指标
- 结果表、得分表、竞赛表、项目清单表中的敏感结果值

## 客户名称处理

- 先按已知客户名单做直接替换
- 再按组织名样式兜底判断
- 典型可识别关键词：
  - `有限公司`
  - `股份有限公司`
  - `银行`
  - `分行`
  - `管理局`
  - `工作部`
  - `委员会`
  - `集团`
  - `政府`
  - `学校`
  - `大学`
  - `医院`

## 使用方法

在 PowerShell 中运行：

```powershell
python d:\hermes\.trae\skills\skill_creptoout\skill_creptoout.py `
  "C:\path\input.pptx" `
  "C:\path\output_redacted.pptx" `
  --verify
```

## 输出说明

- 正常完成会输出：
  - `files_touched`
  - `text_nodes_changed`
  - `chart_numfmt_set`
- 加 `--verify` 会额外输出剩余包含数字的文本节点数量，便于人工抽查

## 推荐工作流

1. 先基于原始 PPT 生成脱敏版
2. 抽查标题、目录页、图表页、客户清单页
3. 如果有误判，优先补规则，再重新生成
4. 通过后再发送给微信或准备上传 GitHub

## 注意事项

- 该脚本当前只处理 `.pptx`，不处理 `.ppt`
- 图表底层数值会尽量保留结构，优先脱敏展示层
- 规则基于当前项目中的政企经营类 PPT 场景沉淀，迁移到新行业时需要再校准
